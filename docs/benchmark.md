# Benchmark methodology

This document describes exactly how the numbers in the [README](../README.md) table were produced,
so that anyone can reproduce (or challenge) them. The frozen result table is in
[benchmark-results.md](benchmark-results.md); the machine-readable source is
`results/baseline/metrics.json` (git-ignored, regenerated by the commands below).

## Test set: 191 images, 8 categories

The manifest is pinned in git at `data/testset/manifest.jsonl` (one JSON line per image:
`id`, `image`, `category`, `gt_alpha`). Category composition:

| Category | n | Source | Notes |
|---|---|---|---|
| hair | 40 | P3M-10k, **P3M-500-NP validation split** | faces obfuscated by the dataset authors |
| thin | 36 | DIS5K **DIS-VD split** | class assigned from the DIS5K class token encoded in each filename (`scripts/build_testset.py::classify_disvd`) |
| complex | 29 | DIS5K **DIS-VD split** | same tokenized classification; "complex" is the default class |
| transparent | 25 | Transparent-460 **test split** | glass, membranes, smoke |
| camouflage | 25 | CAMO **test split** (HF `nobg/camo`) | COD10K's official pixel GT is only distributed via Google Drive with no usable HF mirror, so the CAMO test split (Le et al. 2019) is used instead; the substitution is documented in `scripts/build_testset.py` |
| text | 12 | synthetic, `scripts/make_textfx.py` with **seed 777** | training data uses seed 42 — the deterministic per-item RNG (`_item_rng`) makes the two sets fully disjoint |
| fx | 12 | synthetic, `scripts/make_textfx.py` with **seed 777** | generated with the v4-era fx recipe and its GT; the generator was later reworked for v5 *training* data, so fx scores for lucida-v5 are conservative |
| illustration | 12 | ToonOut **test split** | training used only the ToonOut train split; the test split was deliberately never downloaded during training data preparation (`training/v4_veri_guncelleme_hucresi.py`) |

Two properties matter here:

1. **No train/test leakage.** Every category comes from a held-out split of its source
   (validation/test splits, seed-disjoint synthetic generation, untouched ToonOut test split).
2. **The DIS-VD relabeling is honest history.** The first version of the manifest assigned
   thin/complex labels randomly; this was caught in review and fixed once with
   `scripts/relabel_disvd.py` (categories now derive deterministically from DIS5K filename
   tokens). All published numbers postdate the fix.

## Metrics

Implemented in `benchmark/metrics.py`. Contract: prediction and GT are float32 alpha maps in
`[0, 1]` at the original image resolution.

- **MAE** — mean absolute error. The primary metric in all tables.
- **MSE** — mean squared error.
- **SAD** — sum of absolute differences, divided by 1000 (literature convention).
- **Grad** — gradient error (squared difference of Gaussian gradient magnitudes, sigma 1.4),
  divided by 1000. Uses SciPy's Gaussian derivative — internally consistent, but **not**
  bit-comparable with MATLAB-based published numbers.
- **Conn** — connectivity error (threshold-sweep largest-component formulation), divided by 1000.

Per-category numbers are unweighted means over the images of that category; OVERALL is the
unweighted mean over all 191 images (so categories with more images weigh more).

## Running it

All steps are idempotent: existing alpha PNGs are not recomputed, `metrics.json` entries of models
not in the current run are preserved (merge semantics, `benchmark/run.py::_merge_metrics`), and
Ideogram API calls are skipped when the output file already exists.

```bash
# 1) Open models + local checkpoints -> alpha PNGs + RGBA previews + metrics.json
uv run python -m benchmark.run \
    --models lucida-v5,inspyrenet,rmbg-2.0,birefnet-hr \
    --manifest data/testset/manifest.jsonl \
    --out results/baseline --rgba

# 2) Ideogram commercial reference (fal.ai, ~$0.01/image, capped at 250/run).
#    Fetch is skipped for any image that already has an output file.
export FAL_KEY=...   # https://fal.ai/dashboard/keys
uv run python scripts/fetch_ideogram.py
uv run python scripts/score_ideogram.py     # merges "ideogram" into metrics.json

# 3) Detailed per-metric Markdown comparison (MAE/SAD/MSE/Grad/Conn with deltas)
uv run python scripts/compare_v1.py \
    --metrics results/baseline/metrics.json \
    --v1 lucida-v5,lucida-v5+refine \
    --baselines inspyrenet,ideogram,rmbg-2.0,birefnet-hr

# 4) Optional: side-by-side HTML gallery (original | GT | models | ideogram)
uv run python -m benchmark.gallery \
    --manifest data/testset/manifest.jsonl \
    --results results/baseline \
    --models lucida-v5,inspyrenet,rmbg-2.0,birefnet-hr \
    --out results/baseline/gallery.html
```

Notes:

- `lucida-v5` resolves via `bgr/registry.py` to the local checkpoint
  `data/checkpoints/epoch_5.pth` (download from
  [egeorcun/lucida](https://huggingface.co/egeorcun/lucida)); it runs the BiRefNet_HR architecture
  at 1024px input.
- Model outputs are compared **without** the optional refine/decontaminate post-processing unless
  the model name carries the `+refine` suffix; decontamination only affects RGBA color, never the
  alpha, so it is metric-invariant by construction.

## How the Ideogram reference was obtained

`benchmark/ideogram.py` uploads each test image to fal.ai and calls the
`fal-ai/ideogram/remove-background` endpoint, storing the returned RGBA PNG under
`results/ideogram/<id>.png`. Since Ideogram is an external API rather than a `bgr` segmenter, it
cannot go through `benchmark.run`; `scripts/score_ideogram.py` computes the same metrics from the
stored RGBA alpha channels and merges them into `metrics.json` under the model name `ideogram`,
using the same `_load_alpha`/`_merge_metrics` code paths (imported, not copied). Images whose API
call failed are reported on the console and excluded from that model's averages.
